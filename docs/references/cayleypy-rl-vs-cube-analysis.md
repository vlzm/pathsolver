# Почему вторая статья (CayleyPy RL) добивается лучших результатов, и при чём тут RL

Анализ двух статей серии CayleyPy:

- **Статья 1** — *A Machine Learning Approach That Beats Large Rubik's Cubes* (arXiv:2502.13266). Диффузионный (не-RL) подход + beam search, фокус на кубиках Рубика.
- **Статья 2** — *CayleyPy RL: Pathfinding and reinforcement learning on Cayley graphs* (arXiv:2502.18663). Та же база + новый ингредиент RL/DQN + расширение на LRX-графы Sₙ и математические результаты.

Отчёт отвечает на два связанных вопроса:
1. Помогает ли RL решать лучше, чем без RL?
2. Если вторая статья решает «лучше/дальше», то за счёт чего именно (кроме RL)?

---

## 0. Что в статьях называется «с RL» и «без RL»

Сам поиск пути (**beam search**) — общий для обоих подходов. Различие только в том, **как обучается нейросеть-эвристика** (оценка расстояния до цели):

- **Без RL — «diffusion distance»** (подход из статьи 1): сеть учат предсказывать число шагов случайного блуждания. Данные генерировать дёшево, но есть теоретический изъян.
- **С RL — DQN / уравнение Беллмана**: сеть приближает *истинное* расстояние на графе. Точнее, но дороже по вычислениям.
- **Новый метод статьи 2** — комбинация: warm-up диффузией + Modified DQN.

База (диффузия + beam search) присутствует в **обеих** статьях. RL — это единственная методологическая новинка второй статьи в части алгоритма поиска.

---

## 1. Помогает ли RL? Да, но скромно; сам по себе — плохо

### Прямые утверждения статьи

- **Мотивация (Раздел 2.4):** у чисто диффузионного подхода есть принципиальный недостаток — «diffusion distance is not monotonically dependent on the true distance», и увеличение обучающей выборки не всегда улучшает модель (стагнация, особенно на LRX). Именно это — повод добавить DQN, приближающий истинное расстояние.
- **Стр. 13:** «Additional training by DQN *always* improves the preliminary warm-up training by diffusion distance procedure.»
- **Стр. 12 (важная оговорка):** «We observe stable improvement... comparing to diffusion distance approach, *unfortunately the effect is not so big*.»

### Эксперимент — Таблица 1 (корреляция с истинным расстоянием)

Спирменова корреляция предсказаний сети с истинным расстоянием (эпохи = диффузия + DQN):

| n | 5+0 (без RL) | 5+50 (+RL) | 50+0 (без RL) | 50+50 (+RL) |
|---|---|---|---|---|
| 6 | 0.955 | 0.99 | 0.99 | 0.99 |
| 8 | 0.954 | 0.98 | 0.969 | 0.982 |
| 10 | 0.883 | 0.939 | 0.923 | 0.938 |

→ добавление DQN поднимает корреляцию.

### Эксперимент — Таблица 2 (S₂₈ LRX, длина решения; «None» = путь не найден)

| Эксперимент | 30+200 (диффузия+DQN) | 30+0 (только диффузия) | 100+0 (только диффузия, дольше) | 0+230 (чистый DQN) |
|---|---|---|---|---|
| Медиана | **402** | 650 | 516 | 455.5 |

Идеальная длина ≈ 378. Вывод статьи: «additional training by DQN improves the results — both in length and probability to find a path. **Pure DQN training is not performing well.**» (чистый DQN 0+230 дал None в 7 из 10 запусков).

### Вывод по RL

- Формально: **без RL решается чуть хуже**, чем диффузия+DQN (Табл. 1–2).
- Но эффект **небольшой**, а **чистый RL работает плохо**. Выигрывает не RL сам по себе, а комбинация с диффузионным warm-up.
- Флагманские результаты проекта получены **не** за счёт RL (см. ниже).

## 2. Статьи работают на разных графах — прямого «лучше/хуже» почти нет

- **Статья 1:** кубики Рубика (3×3×3, 4×4×4, 5×5×5).
- **Статья 2:** новое семейство — LRX-графы Sₙ (генераторы L = левый циклический сдвиг, R = правый, X = обмен позиций 0,1), плюс математика (диаметры, гипотеза OEIS-A186783, границы, распределение Гумбеля и т.д.).

Статья 2 **не** заявляет, что решает кубик Рубика лучше первой. Она направляет ту же ML-машину на новую задачу и толкает её по масштабу. Поэтому «во второй решают лучше» правильнее читать как «во второй решают *новое и дальше по масштабу*», а не «тот же бенчмарк с лучшим числом».

---

## 3. Что реально двигает масштаб во второй статье (и это НЕ RL)

По убыванию важности:

### а) «X-condition» — одна строка кода (Раздел 3.1) — главный скачок

```python
if action == 'X' and current_state[0] < current_state[1]:
    continue
```

С теми же пайплайнами это подняло достижимый размер с n≈30 до **n=100** (графы 10³⁰⁺ → 10¹⁵⁰⁺). Логика простая: если позиции 0 и 1 уже отсортированы, применять X незачем. Работает одинаково для всех моделей (перцептрон, бустинг, CNN).
**Оговорка авторов:** это **prior knowledge, специфичное для LRX** («not completely zero prior knowledge»), а не общий ML-приём; к тому же оно мешает находить оптимальные пути при малых n.

### б) Оптимизированная PyTorch-реализация + большой beam search (Разделы 1.2, 2.3)

Оригинальный код для параллельной обработки перестановок, эффективный хэшинг, extraction уникальных состояний — поддержка beam size в **миллионы узлов** (до 2²⁰). Длина решения улучшается почти **линейно по log(beam size)**. То есть больший beam = лучше — это инженерия/поиск, а не обучение. Beam search также решает проблему локальных минимумов, в которых застревает greedy.

### в) Non-backtracking random walks (Раздел 3.3, Таблица 6)

Улучшённая генерация обучающих данных. Эффект сильный:

| Beam Width | Non-backtracking RW | Simple RW |
|---|---|---|
| 2¹⁰ | None | None |
| 2¹² | 252 | None |
| 2¹⁴ | 202 | None |
| 2¹⁶ | 194 | None |
| 2¹⁸ | 194 | None |
| 2²⁰ | 190 (Ideal) | None |

Без non-backtracking single-epoch модель **вообще не решает**. Причина: число шагов non-backtracking-блуждания ближе к истинному расстоянию и имеет меньше шума в обучающих данных.

### г) RL/DQN

Как в разделе 1 — добавляет по краю немного, сам по себе слаб.

## 4. Честная деталь из самой статьи

Для *zero-prior-knowledge* случая (без X-condition) авторы прямо пишут (стр. 14): почти все подходы «**stuck slightly above n=30 — all in the same region, with small variations**». То есть сам обученный эвристик драматически лучше не стал. Прорыв по масштабу (до 90–100) — это **X-condition + большие beam-и**, а не более умная нейросеть.

Отдельно: «overcome the GAP» (Таблицы 4–5) — это сравнение **AI vs классический GAP** на новом LRX-бенчмарке (GAP выдыхается уже при n≈20), а **не** сравнение статьи 2 против статьи 1.

Ещё наблюдение (Раздел 3.3): простые модели (MLP, gradient boosting) работают **лучше**, чем более сложные (трансформеры, CNN); трансформер решал только n ≤ 15.

## 5. Итоговая таблица

| Что нового во 2-й статье | Помогает решать лучше/дальше? | Это ML-прорыв? |
|---|---|---|
| Новое семейство LRX / Sₙ + математика | — (новая область, не прямое сравнение) | нет |
| **X-condition** (1 строка) | **Да, сильнее всего: n=30 → 100** | нет, доменный трюк (prior knowledge) |
| Оптимизированный beam search (миллионы узлов) | Да, заметно | инженерия / алгоритм поиска |
| Non-backtracking random walks | Да (качество данных) | приём генерации данных |
| RL / DQN | Чуть-чуть | скромно |

---

## Короткий ответ

Да, вторая статья дотягивается до заметно бо́льших задач, чем первая, **но не за счёт RL и не за счёт «лучшей нейросети»**. Основные драйверы:

1. **X-condition** — доменный трюк, дающий главный скачок масштаба (n=30 → 100);
2. **более мощная реализация beam search** (миллионы узлов, выигрыш линейный по log beam size);
3. **улучшенная генерация данных** non-backtracking-блужданиями.

То есть выигрыш идёт от **поиска, инженерии и prior knowledge**, а не от обучения эвристики. RL/DQN даёт лишь скромную надбавку поверх диффузионной базы, а сам по себе (чистый DQN) работает плохо.
